# LLM管理功能说明文档

## 功能概述

**功能名称**：LLM管理  
**访问路径**：`/admin/llm-configs`  
**功能描述**：管理大语言模型API配置

## 页面布局

### 页面标题
- **主标题**：LLM管理
- **副标题**：管理大语言模型API配置

### 功能按钮
1. **添加配置**：添加新的大语言模型API配置

### 数据表格
表格显示以下列：
- **提供商**：大语言模型的提供商（如：OpenAI、阿里云、百度等）
- **模型**：具体的模型名称（如：GPT-4、通义千问等）
- **API端点**：API的访问地址
- **状态**：配置的状态（激活/停用）
- **操作**：对配置进行编辑、删除、测试等操作

### 空状态提示
当没有数据时，显示：
- 文字："暂无数据"

## 核心功能

### 1. 添加配置
**触发方式**：点击"添加配置"按钮

**表单字段**（推测）：
- **提供商**（必填）：选择或输入LLM提供商
  - 选项可能包括：OpenAI、阿里云、百度、腾讯等
- **模型名称**（必填）：输入或选择具体的模型名称
- **API端点**（必填）：输入API的访问地址
- **API Key**（必填）：输入API密钥
- **API Secret**（可选）：输入API密钥（如果需要）
- **配置名称**（可选）：为配置设置一个名称
- **状态**（默认）：激活
- **其他配置**（可选）：JSON格式的额外配置参数

**操作按钮**：
- **取消**：关闭弹窗，不保存
- **保存**：保存配置信息
- **测试连接**：测试API配置是否可用

### 2. 配置管理
**编辑配置**：
- 修改已存在的API配置信息
- 更新API密钥等敏感信息

**删除配置**：
- 删除不需要的API配置（需确认）

**启用/停用**：
- 切换配置的激活状态
- 停用的配置不会被使用

**测试配置**：
- 测试API连接是否正常
- 验证API密钥是否有效
- 检查API端点是否可访问

### 3. 数据展示
- 以表格形式展示所有LLM配置
- 显示配置的基本信息和状态
- 支持查看配置的详细信息

### 4. 配置使用
- 激活的配置可以被系统中的AI功能使用
- 可能支持配置的优先级设置
- 可能支持负载均衡或故障转移

## 使用场景

1. **AI功能配置**：为系统的AI功能配置大语言模型
2. **多模型支持**：配置多个不同的LLM提供商和模型
3. **API管理**：管理API密钥和访问配置
4. **故障切换**：当某个API不可用时，切换到备用配置
5. **成本控制**：通过启用/停用配置控制API使用

## 注意事项

1. API密钥是敏感信息，需要妥善保管
2. 删除配置前需要确认，避免误操作
3. 配置被使用时，删除或停用操作可能影响系统功能
4. API端点地址需要正确，否则无法连接
5. 不同提供商的API格式可能不同，需要正确配置
6. 建议定期测试API配置，确保可用性

## 常见LLM提供商

- **OpenAI**：GPT-3.5、GPT-4等
- **阿里云**：通义千问等
- **百度**：文心一言等
- **腾讯**：混元等
- **其他**：根据系统支持情况

## 技术说明

- 配置信息可能加密存储
- API调用可能需要代理或特殊网络配置
- 支持配置的优先级和负载均衡
- 可能支持API调用的监控和日志记录
